Timestamp Microservice

This project fulfills the following user stories :

  1. The API outputs a JSON object in the form of {"unix": <date.getTime()>, "utc" : <date.toUTCString()> }
  2. A valid date string contained in the project url can either be a unix timestamp in milliseconds or an ISO-8601 compliant date string (e.g. "2016-11-20") new Date(date_string).
  3. If the date string is empty, the current timestamp is used as the date.
  4. If the date string is invalid, the api returns a JSON having the structure
    {"unix": null, "utc" : "Not a Date" }.
